GAPS FOUND
================================================================================

CRITICAL GAP #1: Limited Parameter Visibility
  Problem: Instructor UI hardcoded to show only ~30 parameters (20% coverage)

  Missing from display:
    - Per-panel solar array currents (6 panels: +X, -X, +Y, -Y, +Z, -Z)
    - Per-power-line currents (8 lines: OBC, RX, TX, payload, cooler, heaters, wheels)
    - Power line on/off state (switchable vs unswitchable)
    - Battery depth-of-discharge (DoD%), charge cycles
    - Load shedding stage (0-3)
    - Solar array degradation factors
    - Solar array lifetime hours in sunlight
    - MPPT efficiency
    - Reaction wheel speeds (individual RW 1/2/3/4)
    - Attitude quaternion [q1, q2, q3, q4]
    - Star tracker status (both units: OFF/BOOTING/TRACKING/BLIND/FAILED)
    - CSS (coarse sun sensor) sun vector
    - Magnetometer field vector (dual mag, active selection)
    - Gyroscope bias estimates
    - Magnetorquer duty cycles (X/Y/Z)
    - Per-zone temperatures (6 panel faces, OBC, battery, FPA, thruster)
    - Heater duty cycles (%, not just on/off)
    - Heater setpoints
    - OBDH CPU load, RAM, task count, reboot count
    - OBDH HK buffer fill level
    - OBDH active OBC unit (A or B)
    - OBDH active CAN bus (A or B)
    - GPS state (fix type, PDOP, satellite count)
    - Spacecraft phase (0-6: pre-sep, sep, init-pwr, bootloader, leop, commissioning, nominal)
    - FDIR load shedding stage
    - Failure list with magnitude and onset type
    - TM store fill levels (individual stores)
    - Link margin (dB above minimum)
    - All 6 orbital elements (a, e, i, RAAN, ω, ν)
    - Payload filter wheel position
    - Payload calibration buffer state
    - Payload shutter position

CRITICAL GAP #2: RF Link Gating in Instructor Display
  Problem: Original endpoints respected downlink_active flag
  Impact: Instructor couldn't see state during LEOP/link-down phases
  Root: /api/state and /api/mcs-state both applied link gating

CRITICAL GAP #3: Subsystem Internal State Not Exposed
  Problem: Model dataclasses had full internal state but no JSON serialization
  Impact: Deep diagnostics impossible (e.g., "why is wheel 2 slower than wheel 3?")

  EPSState has 30+ fields; only 7 were shown
  AOCSState has 20+ fields; only 6 were shown
  TCSState has 15+ fields; only 6 were shown
  OBDHState has 10+ fields; only 3 were shown
  TTCState has 8+ fields; only 5 were shown
  PayloadState has 8+ fields; only 3 were shown

CRITICAL GAP #4: No Test Coverage
  Problem: No tests verified instructor had complete visibility
  Impact: Gaps went undetected; no regression tests
