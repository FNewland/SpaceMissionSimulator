REGRESSION 1: ON-DEMAND TM LEAK (FIXED)
────────────────────────────────────────
Root Cause: downlink_active property checked TTC link status (0x0501) but
            NOT orbital contact (_in_contact), unlike uplink_active

Impact: S20.3 GET responses leaked out of contact. A spacecraft outside
        orbital view with healthy transponder could still send TM if an
        instructor override was active.

Root Source: packages/smo-simulator/src/smo_simulator/engine.py:641-654
             Original downlink_active only checked: bool(params[0x0501])
             Correct downlink_active checks: (_in_contact AND link_ok) OR override

Fix Applied:
  File: engine.py:656
  Changed: return bool(link_status)
  To: return ((self._in_contact and bool(ttc_link_ok)) or self._override_passes)

Regression Test: Added 4 new tests in test_link_gating.py:394-532
  ✓ test_s20_get_blocked_out_of_contact: Verifies block when out of contact
  ✓ test_s20_get_blocked_link_failed: Verifies block when link down
  ✓ test_s20_get_allowed_contact_and_link_ok: Verifies allow when both OK
  ✓ test_s20_get_allowed_override: Verifies override bypass

Status: ALL TESTS PASSING (1085/1085, 1 skipped)
