8. Cascading Failure Chains

Six critical cascading chains have been identified where a single failure propagates across multiple subsystems:

Chain 1: Solar Array Total Loss → Catastrophic Load Shedding

solar_array_total_loss → power_gen = 0W (no solar input) → bat_soc drops ~2%/min under load → SoC < 30%: LOAD_SHED_STAGE_1 → Payload OFF → SoC < 20%: LOAD_SHED_STAGE_2 → TTC TX OFF, AOCS wheels OFF → SoC < 10%: LOAD_SHED_STAGE_3 → All non-essential OFF → AOCS forced to MODE_ECLIPSE (gyro-only propagation) → Spacecraft in safe/beacon mode until solar recovery

Chain 2: Multi-Wheel Seizure → Attitude Control Loss

rw_seizure[wheel=0] + rw_seizure[wheel=1] → active_wheels < 3 → MODE_FINE_POINT → MODE_NOMINAL (fallback) → MODE_NOMINAL → MODE_COARSE_SUN (if <3 wheels) → Fine-resolution imaging impossible → Payload mode forced to STANDBY → Mission science capability lost

Chain 3: OBC Crash → Bootloader Revert

obc_crash OR memory_corruption → _reboot(REBOOT_WATCHDOG) → sw_image = SW_BOOTLOADER → engine._tick_spacecraft_phase detects sw_image=0 → _enter_bootloader_mode() → Phase → BOOTLOADER_OPS (phase 3) → All HK SIDs disabled (except SID 11 beacon) → Application commands rejected → If boot_image_corrupt: STUCK indefinitely

Chain 4: Thermal Overtemp → Multi-Subsystem Shutdown

obc_thermal[30W] + cooler_failure → OBC temp rises → battery temp rises (thermal coupling) → S12 param 0x0407 (bat_temp) exceeds limit → S19 rule 3002 → S8 func 12 (transponder TX off) → Downlink lost → S12 param 0x0406 (obc_temp) exceeds limit → S19 rule 3003 → S8 func 40 (OBDH safe mode) → Limited operations only

Chain 5: Uplink Loss → Autonomous-Only Operation

uplink_loss OR antenna_deploy_failed → carrier_lock = False → RX_available = False → No TC acceptance → Operator cannot send commands → S12/S19 autonomous rules still function → Spacecraft enters safe mode autonomously → Recovery requires physical link restoration

Chain 6: Sensor Cascade → Attitude Degradation

st_failure[unit=1] + css_failure → st_valid = False, css_valid = False → MODE_FINE_POINT → MODE_NOMINAL (line 685-687) → MODE_COARSE_SUN blocked (css_valid required) → Gyro-only propagation; drift accumulates → att_error exceeds S12 threshold → S19 rule 2004 → AOCS safe mode → Fine-pointing and imaging lost

9. FDIR Integration

Three-Level Response:
  • Level 1 (Warning): S5 event generated, no mode change
  • Level 2 (Alert): Forces spacecraft to SAFE mode if currently NOMINAL
  • Level 3 (Critical): Forces spacecraft to EMERGENCY mode immediately
