1. System Overview

3. Quick Start (Single Machine)

Get everything running on one machine in 3 commands:

# 1. Clone the repository (or extract the distribution archive)
git clone https://github.com/FNewland/SpaceMissionSimulator.git
cd SpaceMissionSimulator

# 2. Run the installer
./deploy/install.sh

# 3. Start all services
./start.sh

# 4. Open in your browser:
#    Instructor: http://localhost:8080
#    MCS:        http://localhost:9090
#    Planner:    http://localhost:9091
That's it! For single-machine use with default settings, no configuration changes are needed. All services bind to localhost and use their default ports.

Quick Start with RF Simulation

# Install with RF simulation support
./deploy/install.sh --rfsim

# Start with CCSDS framing + channel coding
./start.sh --rf

# Additional UI available:
#    Cortex RF Status: http://localhost:8094

4. Full Installation Guide

This section walks through every step for a fresh machine with no Python or tools installed.

1 Install System Dependencies

macOS

# Install Homebrew (if not already installed)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install Python 3.12 and git
brew install python@3.12 git

# Verify
python3 --version    # 3.12.x
git --version

Ubuntu / Debian

# Update package lists
sudo apt update

# Install Python 3.11, venv module, and git
sudo apt install -y python3.11 python3.11-venv python3.11-dev git

# Verify
python3.11 --version
git --version

Fedora / RHEL 9+

sudo dnf install -y python3.11 git
python3.11 --version

2 Get the Source Code

# Clone from GitHub
git clone https://github.com/FNewland/SpaceMissionSimulator.git
cd SpaceMissionSimulator

# Or, if you have a distribution archive:
tar xzf smo-suite-*.tar.gz
cd smo-suite-*

3 Run the Installer

# Full installation (all components)
./deploy/install.sh

# Or choose specific install options:
./deploy/install.sh --rfsim          # Include RF simulation
./deploy/install.sh --dev            # Include test dependencies
./deploy/install.sh --rfsim --dev    # Both

The installer will:

  1. Verify Python 3.11+ is available
  2. Create a virtual environment in .venv/
  3. Upgrade pip, setuptools, and wheel
  4. Install all SMO packages in development mode (pip install -e)
  5. Install additional dependencies (sgp4, numpy, aiohttp, etc.)
  6. Configure vendor asset symlinks for the web UIs
  7. Verify all imports work correctly

4 Verify the Installation

# Run the built-in verification
./deploy/install.sh --check

# Or verify manually
source .venv/bin/activate
python -c "import smo_common, smo_simulator, smo_mcs, smo_planner"
echo "All core packages OK"

5 Start the Suite

# Quick start (all services + tools)
./start.sh

# Or use deploy scripts (reads smo-env.conf for network config)
./deploy/start-all.sh

6 Open the UIs

5. Network Architecture

5.1 Single Machine

All components run on the same machine. The MCS connects directly to the simulator's TCP ports.

Machine A (your computer): Simulator → TC :8001, TM :8002, HTTP :8080 MCS → HTTP :9090 (connects to localhost:8001/8002) Planner → HTTP :9091 Delayed TM → HTTP :8092 Orbit Tools → HTTP :8093 Doc Viewer → HTTP :8095 Browser access: http://localhost:9090 ← MCS http://localhost:9091 ← Planner (standard) http://localhost:9091/wide ← Planner (5760×1080) http://localhost:8080 ← Instructor

5.2 Two-Machine Split (Simulator + Ops)

Machine A (Instructor): 192.168.1.100 Simulator → TC :8001, TM :8002, HTTP :8080 Machine B (Operators): 192.168.1.101 MCS → HTTP :9090 (connects to 192.168.1.100:8001/8002) Planner → HTTP :9091 Browsers on network: http://192.168.1.101:9090 ← MCS http://192.168.1.101:9091 ← Planner http://192.168.1.100:8080 ← Instructor

5.3 Full Distribution (3+ Machines)

Machine A (Simulator): 192.168.1.100 Simulator → TC :8001, TM :8002, HTTP :8080 Machine B (MCS): 192.168.1.101 MCS → HTTP :9090 Machine C (Planner): 192.168.1.102 Planner → HTTP :9091 Machine D (Gateway): 192.168.1.103 [optional] Gateway → :10025 (relays to Simulator)

10. RF Simulation Layer

The RF simulation bridge sits between the simulator and MCS, adding realistic CCSDS transfer framing and RF channel effects.

10.1 Installation

# Install with RF simulation support
./deploy/install.sh --rfsim

# Or install manually
source .venv/bin/activate
pip install -e packages/smo-rfsim

10.2 Operating Modes

ModeDescriptionDependencies
PACKETTransparent relay — no framing or channel effectsaiohttp
FRAMECCSDS Transfer Framing: TM frames (1115B, ASM, FECF), TC CLTUs (BCH 64/56), RS(255,223), BER injectionaiohttp, reedsolo
RFFull BPSK modulation/demodulation via GNU Radio flowgraphsaiohttp, reedsolo, GNU Radio 3.10+

10.3 Starting with RF Simulation

# FRAME mode (CCSDS framing, no GNU Radio needed)
./start.sh --rf
# or
./start.sh --rf=FRAME

# RF mode (requires GNU Radio)
./start.sh --rf=RF

# Via environment variable
export SMO_RF_MODE=FRAME
./start.sh

10.4 Cortex Web UI

When the RF bridge is running, a real-time status dashboard is available at http://localhost:8094, showing:

  • Current Eb/N0 and BER
  • Frame sync state (SEARCH → VERIFY → LOCK)
  • Frame counters and error rates
  • Virtual channel utilization

10.5 Configuration

RF simulation is configured in configs/eosat1/rfsim.yaml. Key settings:

# Channel quality
channel:
  eb_n0_db: 10.0        # Higher = cleaner signal
  ber_target: 1.0e-6    # Target bit error rate
  awgn_enabled: true    # Additive white Gaussian noise

# CCSDS framing
ccsds:
  rs_enabled: true      # Reed-Solomon forward error correction
  fecf_present: true    # Frame Error Control Field (CRC-16)

17. Component Reference

17.1 Simulator

PUS-C service stack: Services 1, 3, 5, 6, 8, 9, 11, 12, 15, 17, 19, 20.

  • Entry point: smo-simulator --config configs/eosat1/ [--speed 1.0]
  • Instructor UI: Inject failures, run scenarios, set breakpoints, control sim speed
  • Subsystems: 6-panel solar array, dual magnetometers, 6 CSS heads, 2 star cameras, battery-heater thermal, multispectral payload (4 bands: 443/560/665/865 nm)
  • State machines: Separation (7 phases), bootloader, transponder command channel

17.2 MCS (Mission Control System)

WCAG 2.1 AA-compliant web UI with 6 operator positions and 11 subsystem tabs.

  • Entry point: smo-mcs --config configs/eosat1/ --connect <host>:<port> --port 9090
  • Tabs: Overview, EPS, AOCS, TCS, OBDH, TTC, Payload, Commanding, PUS, Procedures, Manual
  • Features: Real-time TM via WebSocket, SVG block diagrams, Chart.js trending, alarm journal (S5/S12), procedure execution, GO/NO-GO coordination, world map, stored TM playback (S15), contact timeline (24h, dual ground station)

17.3 Planner

Mission planning with SGP4 orbit propagation.

  • Entry point: smo-planner --config configs/eosat1/ --port 9091
  • Features: Contact window prediction (Iqaluit + Troll ground stations), pass-based scheduling with conflict detection, power budget (SoC prediction), data budget (downlink capacity), imaging target planner (7 ocean current targets)

17.4 Gateway

  • Entry point: smo-gateway --upstream <host>:<port> --listen 0.0.0.0:10025
  • TM: Fans out to all connected downstream clients
  • TC: Forwards from any downstream client to the upstream simulator

17.5 RF Simulation Bridge

  • Entry point: smo-rfsim --config configs/eosat1/rfsim.yaml --mode FRAME [--cortex-web]
  • CCSDS Transfer Framing: TM frames (1115B, ASM, FECF), TC CLTUs (BCH 64/56)
  • Channel coding: RS(255,223) via reedsolo, rate-1/2 K=7 convolutional + Viterbi
  • Channel model: Eb/N0-based BER injection, AWGN, Doppler, path loss

18. Troubleshooting

18.1 "Command not found: smo-simulator"

# Make sure the virtual environment is activated:
source .venv/bin/activate

# Re-install the package:
pip install -e packages/smo-common -e packages/smo-simulator

18.2 "Address already in use"

# Another process is using the port. Find and stop it:
lsof -i :8080    # macOS/Linux
kill <PID>

# Or change the port in deploy/smo-env.conf

18.3 MCS cannot connect to Simulator

  • Verify the simulator is running: curl http://<sim-host>:8080/api/state
  • Check SIMULATOR_HOST in smo-env.conf
  • Check firewall rules (ports 8001, 8002 must be open)
  • Try telnet <sim-host> 8002 to verify TCP connectivity

18.4 "WebSocket connection failed"

  • The MCS falls back to HTTP polling automatically
  • Corporate proxies may block WebSocket — try direct network access

18.5 Python version error

# If you get "requires-python >= 3.11" errors:
python3 --version   # Must be 3.11+

# Recreate the venv with the right Python:
rm -rf .venv
python3.11 -m venv .venv
./deploy/install.sh

18.6 "ModuleNotFoundError: No module named 'venv'"

# Ubuntu/Debian: install the venv module separately
sudo apt install python3.11-venv

# Then retry the installation

18.7 pip install fails with build errors

# Upgrade build tools first:
source .venv/bin/activate
pip install --upgrade pip setuptools wheel

# Then retry:
pip install -e packages/smo-common

18.8 Vendor assets / UI looks broken

# Fix vendor symlinks:
bash fix_vendor_symlinks.sh

# If vendor/ directory is missing, the UI will still work
# but will use fallback styling

18.9 RF Bridge: "reedsolo not found"

source .venv/bin/activate
pip install reedsolo>=1.7

18.10 Tests fail with import errors

# Make sure all packages are installed:
source .venv/bin/activate
pip install -e packages/smo-common
pip install -e packages/smo-simulator -e packages/smo-mcs
pip install -e packages/smo-planner -e packages/smo-gateway
pip install -e packages/smo-rfsim
pip install pytest pytest-asyncio

18.11 Wide-screen planner doesn't fill the screen

  • Use full-screen mode (F11)
  • Set browser zoom to 100% (Ctrl+0 / Cmd+0)
  • The layout adapts to available width

20. Uninstallation

The SMO Suite installs everything locally within the project directory. To remove:

# Remove the virtual environment (all Python packages)
rm -rf .venv
rm -rf .venv-rfsim  # if present

# Remove runtime data
rm -rf workspace/

# Remove the entire project
cd ..
rm -rf SpaceMissionSimulator

No system-wide files are modified. No root/admin privileges are needed for installation or removal.
